# Recovery and Continuation

Use this methodology after preemption, death, restart reconciliation, mobility
failure, or a tool result that leaves the task incomplete.

1. Re-read task/checkpoint and authoritative Body state. Treat pre-crash or
   preemption-era action results as stale until world truth confirms them.
2. Classify the blocker from the typed result: transient transport, owner
   preemption, mobility/approach, missing prerequisite, governance refusal, or
   terminal impossibility.
3. Search durable memory for the place, obstacle, or previously successful
   method when that knowledge could change the next attempt. Re-observe mutable
   world facts before acting on old spatial memory.
4. Resume the existing objective after a Body reflex completes. Do not create a
   competing action while the reflex or another owner still controls the body.
5. Select a materially different approach only when evidence supports it.
   Repeating the same target and parameters is not a strategy change.
6. Record a checkpoint with evidence after real progress. If no viable next
   action exists, yield with the exact blocker instead of extending an internal
   retry loop.
7. Write long-term memory only for stable facts or reusable experience; routine
   errors already belong in observation history.

Recovery never bypasses player-block governance or authoritative terminal truth.
